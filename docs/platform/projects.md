# Projects

**Projects** is where planning and delivery come together. Each project bundles its brief,
specs, development plan, and live status in one place — and Devplan keeps the status current
automatically from the work actually happening.

## Why it matters

Project status is usually stale the moment it's written. Devplan generates **automated project
status** from your commits, meetings, chat, and tickets — no manual reporting. For each project
you can see:

- **Completion** at a glance, derived from real activity
- **Risks and blockers** surfaced early — for example, a PR that's been blocking progress and
  for how long
- **Requirement verification** — whether what's shipping matches what was specced

## What you can do

- Browse and filter projects; toggle done and archived projects in or out
- **Group by release** for a delivery-oriented view
- **+ Add** to create a project from a text description, a document, or ticket links
- Open a project to review its brief, user stories, specs, and build plan, and to track status
  as work progresses
